# sim_data_performance

This topic is for plotting simulation benchmark performance from the
`05_benchmark_analysis` outputs.

## Objective

1. Record how sim data is discovered and organized upstream.
2. Define a stable topic directory layout for plotting work.
3. Prepare one aggregated intermediate table for downstream panel scripts.

## Upstream Data Organization (from `05_benchmark_analysis`)

### Input roots

- Tool prediction results: `data/result/benchmark/sim_bam/{tool}/{sample}/...`
- Ground truth matrix: `data/sim_data/bam/{sample}.bam.expr.tsv`
- Ground truth PAS BED: `data/sim_data/sim_pas/{genome}_sim_pas_{gene}_rep{rep}.bed`

### Sample discovery and filtering logic

- `Snakefile` auto-detects `GENE_TAG` (`gn\\d+`) from `data/sim_data/bam/*.bam.expr.tsv`.
- Candidate samples are built from the intersection of:
  - directories containing `pas_fix.bed` (or `pas.bed`)
  - directories containing `pas_counts.tsv`
- Effective sample key is `"{tool}/{sample_id}"`.
- For DaPars2-source DE-APA evaluation, samples are additionally filtered by:
  - ground-truth sample existence
  - exclude list (`dapars2_single_group_exclude.txt`)
  - required DaPars2 group columns (`Sample_1`/`Sample_2`)

### Output tables used by plotting prep

Under `data/result/performance/{tool}/`:

- `{sample}_match_performance.tsv`
- `{sample}_pas_quantify_performance.tsv`
- `{sample}_de_apa_performance.tsv`

These are generated by `calculate_performance` and
`calculate_dapars2_de_apa_performance` in
`scripts/05_benchmark_analysis/Snakefile`.

## Aggregation Script Design

`scripts/00_prepare_data.py` aggregates the three performance table families
from `data/result/performance/*/` into one topic-level table:

- Add `metric_domain` to distinguish table origin:
  - `match_performance`
  - `pas_quantify_performance`
  - `de_apa_performance`
- Keep all original metric columns (wide union by `pandas.concat(sort=False)`).
- Fill missing `tool` / `sample` / `protocol` from file path when needed.
- Apply `_shared.constants` mapping for display-friendly tool/protocol names.
- Write:
  - `data/intermediate/sim_data_performance_prepared.parquet`
  - `data/intermediate/metadata.json`

This intermediate table is the only required input for future panel scripts.

## Separate Pipeline Aggregation

`scripts/01_prepare_dapars2_pipeline_data.py` handles the separate DaPars2 /
differential APA pipeline outputs:

- `data/result/dapars2_format/*/*_dapars2_format.txt`
- `data/result/dapars2_format_ground_truth/*_dapars2_format.txt`
- `data/result/differential_apa/*/*/*/APA_results.txt`
- `data/result/differential_apa_ground_truth/*/*/APA_results.txt`

Outputs:

- `data/intermediate/sim_data_dapars2_pipeline_manifest.parquet`
  - one row per source file
  - includes parsed `tool`, `sample`, `comparison`, header schema, file size
- `data/intermediate/sim_data_dapars2_pipeline_summary.parquet`
  - grouped summary by pipeline domain and tool/comparison
- `data/intermediate/sim_data_dapars2_pipeline_metadata.json`

Optional:

- `--with-row-count` computes row count for each source file (slower on full
  dataset).

## APA Residual Aggregation

`scripts/02_prepare_apa_residuals.py` integrates DE-APA performance tables and
computes residual features used by reference residual plots.

Inputs:

- `data/result/performance/*/*_de_apa_performance.tsv`

Core derivations:

- `gt_recall = gt_count / gt_count_total` (default `5000`)
- per-group `f1` trimming before regression:
  - keep only `q1-q99` (`--f1-lower-quantile=0.01`, `--f1-upper-quantile=0.99`)
- group by `sample`, `tool`, `match_type`
- linear regression: `f1 ~ gt_recall`
- output:
  - `f1_intercept`, `f1_coefficient`, `f1_r2`, `f1_residuals`
- combine filter keys:
  - `filter_type_comb = filter_type_1 + "|" + filter_type_2`

Outputs:

- `data/intermediate/sim_data_apa_residuals.parquet`
- `data/intermediate/sim_data_apa_residuals_by_filter.parquet`
- `data/intermediate/sim_data_apa_residuals_metadata.json`

## Differential-APA Single-Filter Aggregation

`scripts/03_prepare_differential_apa_single_filter_performance.py` subsets
the 05-benchmark DE-APA performance outputs to one fixed filter combination
for fair cross-tool comparison (including `DaPars2` and `scMAPA`):

Inputs:

- `data/result/performance/*/*_de_apa_performance.tsv`

Core derivations:

- keep rows matching exact `filter_type_1` / `filter_type_2`
  (defaults: `scmapa_adPval_0.05` + `scmapa_or_0.1`)
- preserve benchmark-exported performance metrics directly

Outputs:

- `data/intermediate/sim_data_differential_apa_single_filter_performance.parquet`
- `data/intermediate/sim_data_differential_apa_single_filter_metadata.json`

## Runtime Options

- `--data-root`:
  - Can be either project root (contains `data/`) or the `data` directory itself.
  - Default is the current repository checkout root.
- `--filter-type-1-suffix`:
  - Residual regression pre-filter on `filter_type_1` suffix.
  - Default: `0.05`.
- `--gt-count-total`:
  - Denominator used to compute `gt_recall`.
  - Default: `5000`.
- `--f1-lower-quantile` / `--f1-upper-quantile`:
  - Quantile range used by `02_prepare_apa_residuals.py` for trimming `f1` before regression.
  - Defaults: `0.01` / `0.99`.
- `--include-stage-files`:
  - For `00_prepare_data.py`, include stage outputs (`_stage10_` / `_stage20_`).
  - Default behavior excludes stage files and keeps final outputs only.
- `--max-files-per-domain`:
  - Optional debug cap of input files per metric domain for `00_prepare_data.py`.
  - Default: `0` (use all files).
- `--plot-config`:
  - JSON file for panel rendering overrides (figure size/layout/dpi/boxplot/grid).
  - Default:
    `scripts/06_plotting/sim_data_performance/config/plot_params.json`
  - Forwarded via environment variable `SIM_DATA_PERF_PLOT_CONFIG`.
- `--force-prepare`:
  - Force rerun prepare scripts even when intermediates already exist.
  - Default behavior is to reuse intermediates and only prepare missing outputs.
- `--plots-only`:
  - Skip all prepare scripts and render panels only.
  - Requires existing intermediate files.
- `--with-pipeline-manifest`:
  - Also run `01_prepare_dapars2_pipeline_data.py`.
  - Not required for panel plotting; default is skipped.

Example:

```bash
python scripts/06_plotting/sim_data_performance/scripts/00_prepare_data.py \
  --data-root /path/to/apabenchmark_final
```

```bash
python scripts/06_plotting/sim_data_performance/scripts/01_prepare_dapars2_pipeline_data.py \
  --data-root /path/to/apabenchmark_final
```

```bash
python scripts/06_plotting/sim_data_performance/scripts/02_prepare_apa_residuals.py \
  --data-root /path/to/apabenchmark_final \
  --filter-type-1-suffix 0.05 \
  --gt-count-total 5000
```

Or via workflow entry:

```bash
python scripts/06_plotting/sim_data_performance/scripts/run_all.py \
  --data-root /path/to/apabenchmark_final \
  --plot-config scripts/06_plotting/sim_data_performance/config/plot_params.json \
  --filter-type-1-suffix 0.05 \
  --gt-count-total 5000
```

Default execution strategy:

- required prepare scripts (`00`, `02`, `03`) run only when outputs are missing
- existing intermediate files are reused
- pipeline manifest prepare (`01`) is skipped unless `--with-pipeline-manifest`

## Unified Plot Tuning Interface

Edit this file:

`scripts/06_plotting/sim_data_performance/config/plot_params.json`

Then rerun `run_all.py` once. The panel scripts now read this config for:

- figure size presets (`size_presets_mm`)
- output DPI (`render.savefig_dpi`, `render.per_figure_dpi`)
- subplot spacing/legend anchors (`layout.*`)
- boxplot options (`boxplot.*`)
- grid line style (`grid.*`)
- optional matplotlib `rcParams` overrides (`rcParams`)

## Boxplot Outlier Handling

- Boxplot scripts trim per-metric values to `q1-q99` before plotting.
- Outlier markers are disabled in rendered boxplots (`showfliers=False`).
