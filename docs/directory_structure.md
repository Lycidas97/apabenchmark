# Directory Structure

This repository keeps executable workflow code in git and keeps large runtime
data outside git. The current directory names are part of the v0.1 pre-release
contract because Snakemake rules and wrapper scripts refer to them directly.

## Repository Control Plane

| Path | Purpose |
| --- | --- |
| `README.md` | Release-facing overview and quick start. |
| `docs/` | Active release documentation, manifests, and checklists. |
| `docs/archive/` | Historical notes and legacy examples; not part of the active workflow contract. |
| `scripts/envs/` | Conda environment definitions used by workflow stages. |
| `scripts/apasim/` | Local simulation helper package used by Stage 03. |
| `scripts/run_full_repro.sh` | Top-level connected workflow runner for stages 01-06. |
| `scripts/run_minimal_repro.sh` | Smaller Stage 05/06 runner for precomputed inputs. |

## Workflow Stages

| Path | Stage | Responsibility |
| --- | --- | --- |
| `scripts/01_generate_annotation/` | Stage 01 | Integrate public PAS annotation sources and write filtered annotations. |
| `scripts/02_peak_analysis/` | Stage 02 | Process empirical BAM inputs and estimate peak-shape summaries. |
| `scripts/03_data_simulation/` | Stage 03 | Generate simulated PAS sets, simulated BAMs, and CPRSB inputs. |
| `scripts/04_benchmark/` | Stage 04 | Run benchmarked APA tools and collect tool outputs/resource profiles. |
| `scripts/05_benchmark_analysis/` | Stage 05 | Match predicted PAS, quantify performance, and run differential APA analysis. |
| `scripts/06_plotting/` | Stage 06 | Prepare plotting intermediates and render figure panels. |

The numbered stage directories are intentionally stable. Renaming them before
release would require coordinated updates across shell entrypoints, Snakemake
files, wrapper scripts, documentation, and ignored runtime paths.

## Data Plane

The `data/` directory is ignored by git. It must be supplied or generated beside
the repository checkout for full reproduction.

| Path | Meaning |
| --- | --- |
| `data/raw_data/` | External inputs and prepared non-git payloads, including annotations, empirical BAMs, raw benchmark inputs, and tool references. |
| `data/int_data/` | Intermediate outputs produced by upstream stages, including integrated annotations, processed BAMs, peaks, and model summaries. |
| `data/sim_data/` | Simulated PAS, BAM, expression, and CPRSB input files generated by Stage 03. |
| `data/result/` | Benchmark, analysis, plotting source, and resource-profile outputs from downstream stages. |

These names are legacy-compatible and are kept as the v0.1 release contract.
Future releases can add clearer aliases or migration scripts, but the current
release should not rename them in place.

## Stage 04 Subdirectories

| Path | Purpose |
| --- | --- |
| `scripts/04_benchmark/sif/` | Local destination for downloaded Singularity/Apptainer images. The images are ignored by git. |
| `scripts/04_benchmark/snakemake_profile/` | Snakemake profile files and generated sample config location. |
| `scripts/04_benchmark/tool_wrappers/` | Tool-specific wrapper scripts used by the Stage 04 Snakefile. |

## Stage 06 Plotting Topics

Each plotting topic follows the same internal layout: `scripts/` for data prep
and panel scripts, `reference/` for retained notebooks/reference material,
`figures/` for generated outputs, and optional `config/` for panel settings.

| Topic directory | Meaning |
| --- | --- |
| `sim_data_performance/` | Simulation benchmark performance across tools and protocols. |
| `raw_data_performance/` | Optional raw-data benchmark and paired-validation plots. |
| `pas_detect_performance/` | PAS detection-oriented performance panels. |
| `pas_quantification_performance/` | PAS quantification-oriented performance panels. |
| `apa_detect_performance/` | Differential APA detection-oriented performance panels. |
| `peak_model_params/` | Peak model and coverage-shape parameter panels. |
| `computation_resource_consumption/` | Runtime, memory, and CPU resource panels. |
| `_shared/` | Shared plotting constants, path helpers, I/O, and style utilities. |

Topic directory names are treated as stable workflow paths for the v0.1
pre-release because runners, tests, configs, and documentation refer to them.
